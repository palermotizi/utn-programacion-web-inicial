formularios:
<form> 
se puede poner un titulo o algo
 label habla de que trata este input (le ponemos for: id del input)
input: usuario ingresa datos (le ponemos id para el label)
verificamos que esten conectados yendo a la web y dar click al label tiene que ir al input
</form>

todos los input deben tener label (a excepcion de los botones como el submit)
sirve y un monton, es importante para los ciegos. regla de accesibilidad. si no la seguimos, la pagina puntua para abajo osea negativo.

para que el escrito aparezca dentro del titulo esta el atributo placeholder se pone en el input

como no tener label puntua para abajo, pero no queremos que se vea el texto fuera del cuadrado de texto:

cuando escribo el dato el label se sube arriba del placeholder y queda asi

label --------> email
placeholder --> email

siempre se utiliza el label si o si, se puede poner transparente asi no se ve.

mete el label dentro del input

dos formas de hacer un boton para enviar:

button type submit: botoncito enviar, escribo yo el enviar (el texto)

input type submit: pone automaticamente enviar, pero no siempre dice enviar, lo puede poner en ingles, frances, o el idioa que esté. el atributo Value="confirmar"(texto a elegir). desventaja: no lo traduce en otro idioma, queda en confirmar.

ambas se recomiendan.

caso particular:
si el boton tiene un icono nos conviene poner el button.
en input no podemos poner el html o el icono

si pongo un boton y no pongo de tipo submit, por estar dentro del form lo toma como submit. por defecto es de este tipo.
